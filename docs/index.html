OpenShift

Deploy OpenShift on bare metal KVM

Two fully automated deployment modes — both with live progress tracking, MAC-based node identification, and credential collection.

Assisted Installer

Cloud-connected

  • Uses Red Hat Assisted Installer API (console.redhat.com)
  • SNO, 3-node compact, or full HA (N control-plane + M workers)
  • Full-ISO discovery — auto-downloaded and cached by fingerprint
  • MAC-based host identification — VM names shown, not UUID fragments
  • Hostname assignment with 3× retry + verification loop
  • n8n-style monitoring dashboard: node grid, operator panel, event feed
  • Per-host stage tracking with 15-minute stuck node detection
  • Auto-recovery for pending-user-action: eject ISO + reboot correct VM
  • Operator availability tracking during finalizing phase
  • Sync, retry, and reset for stuck or failed deployments
  • Credentials survive service restarts (0600 secrets file)
Agent Installer

Air-gapped / Disconnected

  • Uses local openshift-install agent create image
  • No Red Hat cloud account required — fully offline capable
  • Auto-downloads openshift-install + oc binaries for requested version
  • Generates install-config.yaml + agent-config.yaml automatically
  • NMState static IP configuration via nmstatectl (or DHCP fallback)
  • AppArmor-aware ISO placement at /var/lib/libvirt/images/
  • EFI firmware VMs with host CPU passthrough
  • Per-node vCPU / RAM / disk configuration
  • Reset with ISO reuse option (skips 2–5 min rebuild)
  • Save pull secret + SSH key once; reuse across deployments
  • Live kubectl cluster status after install completes

Getting Started

Up and running in minutes

One command installs everything on any Ubuntu server with KVM support. No Ansible, no containers, no agents.

1

Run the one-line installer

Installs QEMU/KVM, libvirt, Python, Node.js, builds the frontend, and registers a systemd service — all unattended.

2

Open the web UI

Navigate to http://your-server:5000 from any browser. Log in with your Linux username and password via PAM.

3

Create virtual machines

Use the VM creation wizard to configure vCPUs, RAM, disks, and network. Pick from the cloud image catalog (Ubuntu, Debian, Rocky, Fedora…) for instant boot.

4

Deploy OpenShift or Kubernetes

Use the OpenShift wizard (Assisted Installer or Agent-based) or the Kubernetes wizard. Configure cluster name, node count, and credentials — the platform handles everything end-to-end.

Installation

Install on Ubuntu 22.04 / 24.04

Requires a bare metal or nested-virt host with KVM support. Tested on Ubuntu 22.04 LTS and 24.04 LTS.

1

Run the one-line installer

Downloads and runs the install script as root — installs all dependencies, builds the frontend, and registers a systemd service.

sudo bash <(curl -fsSL https://raw.githubusercontent.com/zeshaq/vm-manager/main/install.sh)
2

Custom port or branch (optional)

sudo bash install.sh --port 8080 --branch main
3

Open the UI

Visit http://<server-ip>:5000 and log in with your Linux credentials.

4

Check service health

systemctl status vm-manager

Tail logs: journalctl -u vm-manager -f
