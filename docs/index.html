Features

Everything you need to manage VMs

A full-featured web UI for KVM — from basic VM creation to full OpenShift cluster deployment.

🖥️

VM Lifecycle Management

Create, start, stop, pause, and delete KVM virtual machines from a clean web interface. Bulk operations supported.

📊

Real-time Monitoring

Live CPU, memory, and disk stats per VM. Per-host stage tracking during OpenShift installation with stuck node detection.

🖱️

Browser Console

Full VNC console access to any VM directly in the browser via noVNC — no SSH or separate client needed.

☁️

OpenShift Deployment

Deploy SNO, compact 3-node, or full HA 5-node OpenShift clusters using the Assisted Installer or Agent-based installer.

🔐

Kubernetes on KVM

Deploy single-node or multi-node Kubernetes clusters on KVM VMs using cloud-init and automated node provisioning.

🌐

Network Management

Manage libvirt bridges and networks. Static IP assignment for OpenShift nodes with NMState config generation.

OpenShift

Deploy OpenShift on bare metal KVM

Two deployment modes — both fully automated with live progress tracking and auto-recovery.

Assisted Installer Cloud-connected

  • Uses Red Hat Assisted Installer API (console.redhat.com)
  • Full-ISO discovery with real-time download progress
  • Automatic MAC-based node identification
  • Per-host stage tracking and stuck node detection
  • Auto-recovery for pending-user-action nodes
  • Operator availability tracking during finalizing
  • Supports SNO, 3-node compact, 5-node HA

Agent Installer Air-gapped

  • Uses openshift-install agent create image
  • Works fully offline / air-gapped environments
  • PyYAML-based agent-config for correct indentation
  • AppArmor-aware ISO placement for QEMU
  • Per-node CPU / RAM / disk configuration
  • 7-step task table with live duration tracking
  • Auto MAC assignment and NMState static IP config
How it works

Up and running in minutes

Install once on any Ubuntu server with KVM support. Access from any browser on your network.

1

Run the installer

One command installs all dependencies (QEMU, libvirt, Python, Node), clones the repo, builds the frontend, and registers a systemd service.

2

Open the web UI

Navigate to http://your-server:5000 from any browser. Log in with your Linux username and password.

3

Create virtual machines

Use the VM creation wizard to define CPU, RAM, disk, and network. VMs start immediately.

4

Deploy OpenShift (optional)

Use the OpenShift wizard to configure cluster name, node count, and credentials. The platform creates all VMs, downloads the ISO, and monitors installation end-to-end.

Installation

Install on Ubuntu 22.04 / 24.04

Requires a bare metal or nested-virt host with KVM support. Tested on Ubuntu 22.04 LTS and 24.04 LTS.

1

Run the one-line installer

Downloads and runs the install script as root.

sudo bash <(curl -fsSL https://raw.githubusercontent.com/zeshaq/vm-manager/main/install.sh)
2

Custom options (optional)

Override defaults with flags:

sudo bash install.sh --port 8080 --branch develop
3

Open the UI

Visit http://<server-ip>:5000 in your browser. Login with your Linux credentials.

4

Check service status

systemctl status vm-manager

Logs: journalctl -u vm-manager -f
